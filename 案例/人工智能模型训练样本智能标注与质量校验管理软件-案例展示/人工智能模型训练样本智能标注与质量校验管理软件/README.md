# 人工智能模型训练样本智能标注与质量校验管理软件

本项目为 Python 3 + PyQt6 + SQLite 桌面端系统，围绕“样本入库 → 数据体检 → 标签体系 → 智能预标注 → 标注员修正 → 实时质检 → 返工闭环 → 版本冻结 → 训练格式导出 → 质量报告”构建。

## 运行方式

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

首次运行会自动创建 `data/app.db` 并生成演示图片、文本样本、预标注、质检问题、返工单、版本和报告数据。

## 本次界面优化版本说明

本版本已在原项目基础上进行二次优化，重点减少简单 CRUD 页面，增强真实数据生产系统感：

- 主框架按业务链路分组，左侧导航不再像普通后台菜单。
- 数据集驾驶舱增加健康度仪表、流程管线、趋势线、标签分布和风险热力图。
- 样本标注生产工作台增加风险优先队列、提交门禁、AI 建议、返工说明和多层画布控制。
- 质量校验中心增加高风险样本推荐、问题热力图、严重度分布和完整返工闭环操作。
- 样本导入页面增加导入流水线、入库门禁和体检风险摘要。
- 导出中心增加交付门禁、健康度仪表、训练/验证/测试划分和导出留痕。

详细说明见 `docs/界面优化说明.md`。

## 第三次针对截图反馈的优化

- 主窗口默认打开尺寸改为 `1320×820`，并设置最小尺寸，避免在普通笔记本屏幕上首次打开过大。
- 样本标注生产工作台去掉内容区重复标题，保留主窗口顶部唯一页面标题。
- 工作台三栏比例重新调整为“风险队列 / 中央画布 / 右侧质检面板”，并缩短工具栏按钮文案，减少横向裁切。
- 风险热力图和提交门禁组件改为自适应换行显示，修复“Ground Truth”等文字显示不完整的问题。
- 演示数据扩展为 5 个项目、75 条样本：安全帽检测、交通路口检测、工业缺陷检测、客服文本分类、订单表格异常校验。
- 演示图片扩展为 43 张，并包含晴天/雨天交通、施工现场、工业质检等多种场景。
- 数据库样本路径改为相对路径，换电脑解压后也能正常加载演示图片。


## v4 界面修正

- 已移除页面内容区重复大标题，统一保留主窗口顶部标题与说明。
- 各功能页面只在内部保留业务卡片标题，避免出现“顶部标题 + 页面标题”重复。
- 该处理通过 `app/ui/utils.py` 的 `page_root(..., show_header=False)` 默认策略完成，新增页面也会自动遵循。
